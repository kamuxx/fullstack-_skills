# Skill: UI/UX Designer (The Creative)

## Purpose
To ensure the software is not just functional, but **intuitive, accessible, and delightful**. It bridges the gap between the "Requirements" (Skill 03) and the "Frontend Code" (Skill 04/14). It applies design thinking and usability heuristics.

## Context Compatibility (User: Lester Rodriguez)
Expands on the "Interfaces con Angular/React" experience.
-   **Enterprise:** Clean, data-dense dashboards (Tablas, Grids) for CRM/ERP (Sisconti, Factory HKA).
-   **Modern:** "Wow" effects, Glassmorphism, and intuitive flows for new Apps.

## Inputs
1.  **User Stories:** "As a user, I want to filter invocies..."
2.  **Brand Guidelines:** (If available) Colors, Logo.

## Process
1.  **User Flow Mapping:** Visualizing the path: Login -> Dashboard -> Action -> Success.
2.  **Wireframing (Low-Fi):** Structural layout without distractions.
    -   *Tools:* ASCII Diagrams, Mermaid.js User Journey.
3.  **Design System Definition:**
    -   *Typography:* Header vs Body fonts (Inter, Roboto).
    -   *Color Palette:* Primary, Secondary, Semantic (Error/Success).
4.  **Accessibility Check (A11y):** Contrast ratios, readable font sizes (WCAG).

## Outputs
1.  **Wireframe Definitions:** Layout descriptions for the coder.
2.  **Design Tokens:** JSON/Object with colors, spacing, radii.
3.  **Interaction Guidelines:** "Hover states", "Loading skeletons".
